# Common Trigger Errors

## Overview

This document catalogs common error messages generated by OHFY-Core triggers, their root causes, and recommended solutions. Use this reference when debugging integration failures.

## Status Validation Errors

### "Cannot move to a previous status"

**Error Message**: `"Cannot move to a previous status"`

**Trigger**: `TA_Invoice_BU_StatusUpdater` (Order__c BEFORE_UPDATE)

**Cause**: Attempting to set Order__c status to a lower value than current status.

**Status Progression** (must advance forward):
```
New (1) → Scheduled (1) → In Progress (1) → Picking (2) → Loaded (3) →
Out For Delivery (4) → Delivered (5) → Complete (6)
                                        ↓
                                   Cancelled (7)
```

**Example Failure**:
```javascript
// Current status: "Loaded" (3)
invoice.Status__c = "Picking"; // ERROR - can't go backward from 3 to 2
```

**Solutions**:

1. **Remove status update** if not needed:
```javascript
// Don't update status - leave it as is
delete updateData.Status__c;
```

2. **Use bypass** for legitimate backward movement:
```javascript
U_OrderStatusValidationBypass.isBypassEnabled = true;
try {
    invoice.Status__c = "New"; // Allowed with bypass
    update invoice;
} finally {
    U_OrderStatusValidationBypass.isBypassEnabled = false;
}
```

3. **Progress forward** to valid next status:
```javascript
// Current: "Loaded" (3)
invoice.Status__c = "Out For Delivery"; // Valid - progressing forward
```

## Duplicate Detection Errors

### "This item already exists in the order"

**Error Message**: `"This item already exists in the order"`

**Trigger**: `TA_OrderItem_BI_DuplicateChecker` (Order_Item__c BEFORE_INSERT)

**Cause**: Attempting to insert multiple Order_Item__c records with the same Item__c for the same Order__c.

**Example Failure**:
```javascript
// orderItems contains two items referencing same Item__c
[
    { Order__c: "a0X...", Item__c: "a0Y...123" },
    { Order__c: "a0X...", Item__c: "a0Y...123" } // DUPLICATE
]
```

**Solutions**:

1. **Deduplicate before insert**:
```javascript
// Group by Item__c, sum quantities
const itemMap = new Map();
orderItems.forEach(item => {
    const key = item.Item__c;
    if (itemMap.has(key)) {
        // Combine quantities instead of creating duplicate
        itemMap.get(key).Quantity__c += item.Quantity__c;
    } else {
        itemMap.set(key, item);
    }
});
const deduplicatedItems = Array.from(itemMap.values());
```

2. **Use External_ID__c for upsert** to merge duplicates:
```javascript
// Tray Salesforce connector composite request
{
    "allOrNone": false,
    "compositeRequest": [
        {
            "method": "PATCH",
            "url": "/services/data/v62.0/sobjects/Order_Item__c/External_ID__c/order1_item123",
            "referenceId": "item1",
            "body": {
                "Order__c": "@{orderRef}",
                "Item__c": "@{itemRef}",
                "Quantity__c": 5
            }
        }
    ]
}
```

3. **Bypass duplicate check** (use with caution):
```javascript
MetadataTriggerHandler.bypass('TA_OrderItem_BI_DuplicateChecker');
try {
    insert orderItems; // Allows duplicates
} finally {
    MetadataTriggerHandler.clearBypass('TA_OrderItem_BI_DuplicateChecker');
}
```

**Warning**: Allowing duplicates may cause:
- Incorrect inventory calculations
- Double-charging customers
- Confusion in order fulfillment

## Inventory Errors

### "You cannot remove more inventory than is available"

**Error Message**: `"You cannot remove more inventory than is available"`

**Trigger**: `TA_Inventory_BU_PreventNegativeInventory` (Inventory__c BEFORE_UPDATE)

**Cause**: Attempting to set Inventory__c.Quantity__c to a negative value.

**Example Failure**:
```javascript
// Current inventory: 10 units
inventory.Quantity__c = -5; // ERROR - can't be negative
```

**Solutions**:

1. **Check inventory before update**:
```javascript
// Query current inventory
const currentQty = inventory.Quantity__c;
const requested = 15;

if (currentQty >= requested) {
    inventory.Quantity__c = currentQty - requested;
    update inventory;
} else {
    throw new Error(`Insufficient inventory: ${currentQty} available, ${requested} requested`);
}
```

2. **Use allocation logic**:
```javascript
// Create Allocation__c record instead of directly decrementing
{
    "Inventory__c": "inventoryId",
    "Quantity__c": 15,
    "Order__c": "orderId"
}
// Trigger handles inventory reduction safely
```

3. **Bypass for inventory corrections** (admin operations only):
```javascript
MetadataTriggerHandler.bypass('TA_Inventory_BU_PreventNegativeInventory');
try {
    inventory.Quantity__c = -5; // Correction for audit
    update inventory;
} finally {
    MetadataTriggerHandler.clearAllBypasses();
}
```

## Item Activation Errors

### "Cannot deactivate item with existing inventory"

**Error Message**: `"Cannot deactivate item with existing inventory"`

**Trigger**: `TA_Item_BU_StatusChecker` (Item__c BEFORE_UPDATE)

**Cause**: Attempting to set Item__c.Active__c = false when associated Inventory__c records have Quantity__c > 0.

**Example Failure**:
```javascript
// Item has 50 units in warehouse
item.Active__c = false; // ERROR - must have 0 inventory first
```

**Solutions**:

1. **Zero out inventory first**:
```javascript
// Step 1: Create inventory adjustment to zero
{
    "Inventory__c": "invId",
    "Adjustment_Type__c": "Manual Reduction",
    "Quantity__c": -50 // Bring to zero
}

// Step 2: Deactivate item
{
    "Item__c": "itemId",
    "Active__c": false
}
```

2. **Transfer inventory to another item**:
```javascript
// Move inventory to replacement item
update inventoryRecord {
    Item__c = "replacementItemId"
}

// Then deactivate original item
update item {
    Active__c = false
}
```

3. **Bypass for special cases**:
```javascript
MetadataTriggerHandler.bypass('TA_Item_BU_StatusChecker');
try {
    item.Active__c = false;
    update item;
} finally {
    MetadataTriggerHandler.clearAllBypasses();
}
```

## Unit of Measure Errors

### "No transformation setting found for UOM conversion"

**Error Message**: `"No transformation setting found for UOM conversion"`

**Trigger**: `TA_Item_AI_UOMTransformationCreator` (Item__c AFTER_INSERT)

**Cause**: Missing Transformation_Setting__c record for unit conversion (e.g., EA to CS).

**Example Failure**:
```javascript
// Item uses "CS" (case) but no EA→CS transformation exists
{
    "Item__c": "itemId",
    "Unit_of_Measure__c": "CS"
}
```

**Solutions**:

1. **Create transformation setting**:
```javascript
// Insert missing UOM conversion
{
    "From_UOM__c": "EA",
    "To_UOM__c": "CS",
    "Conversion_Factor__c": 12 // 12 eaches per case
}
```

2. **Use standard UOM** if transformation not available:
```javascript
// Default to "EA" (each)
{
    "Item__c": "itemId",
    "Unit_of_Measure__c": "EA"
}
```

3. **Bypass UOM transformation** (rare):
```javascript
MetadataTriggerHandler.bypass('TA_Item_AI_UOMTransformationCreator');
try {
    insert item;
} finally {
    MetadataTriggerHandler.clearAllBypasses();
}
```

## Relationship Errors

### "Order must have delivery route before completion"

**Error Message**: `"Order must have delivery route before completion"`

**Trigger**: `TA_Order_BU_DeliveryRouteValidator` (Order__c BEFORE_UPDATE)

**Cause**: Attempting to set Status__c = "Complete" when Delivery__c (route) is null.

**Example Failure**:
```javascript
{
    "Order__c": "orderId",
    "Status__c": "Complete",
    "Delivery__c": null // ERROR - route required
}
```

**Solutions**:

1. **Associate delivery route first**:
```javascript
// Step 1: Create or assign delivery
{
    "Order__c": "orderId",
    "Delivery__c": "deliveryRouteId"
}

// Step 2: Complete order
{
    "Order__c": "orderId",
    "Status__c": "Complete"
}
```

2. **Use bypass for special cases**:
```javascript
MetadataTriggerHandler.bypass('TA_Order_BU_DeliveryRouteValidator');
try {
    order.Status__c = "Complete";
    update order;
} finally {
    MetadataTriggerHandler.clearAllBypasses();
}
```

## External ID Errors

### "External_ID__c already exists"

**Error Message**: `"DUPLICATE_EXTERNAL_ID: External_ID__c already exists"`

**Trigger**: Salesforce standard duplicate detection (External_ID__c is unique)

**Cause**: Attempting to insert/update record with External_ID__c that already exists.

**Example Failure**:
```javascript
// External_ID__c "shopify_12345678" already exists
{
    "Item__c": "newItemId",
    "External_ID__c": "shopify_12345678" // DUPLICATE
}
```

**Solutions**:

1. **Use upsert instead of insert**:
```javascript
// Tray Salesforce composite request - use PATCH with external ID
{
    "method": "PATCH",
    "url": "/services/data/v62.0/sobjects/Item__c/External_ID__c/shopify_12345678",
    "body": {
        "Name": "Updated Product Name"
    }
}
```

2. **Query first to check existence**:
```javascript
// Check if exists before attempting insert
const existing = await sf.query({
    query: "SELECT Id FROM Item__c WHERE External_ID__c = 'shopify_12345678'"
});

if (existing.length > 0) {
    // Update existing
    update existing[0];
} else {
    // Insert new
    insert newItem;
}
```

3. **Generate unique External_ID**:
```javascript
// Add timestamp or UUID suffix for uniqueness
{
    "External_ID__c": `shopify_12345678_${Date.now()}`
}
```

## Composite Request Errors

### "Cannot insert Order_Item__c: Order__c does not exist"

**Error Message**: `"INVALID_CROSS_REFERENCE_KEY: Order__c does not exist"`

**Cause**: Master-detail relationship violation - attempting to insert child record before parent exists.

**Example Failure**:
```javascript
// Order not yet created but trying to reference it
{
    "compositeRequest": [
        {
            "referenceId": "item1",
            "method": "POST",
            "url": "/services/data/v62.0/sobjects/Order_Item__c",
            "body": {
                "Order__c": "@{order1.id}", // order1 not yet in request!
                "Item__c": "itemId"
            }
        }
    ]
}
```

**Solutions**:

1. **Follow correct composite request order**:
```javascript
{
    "compositeRequest": [
        // 1. Create parent first
        {
            "referenceId": "order1",
            "method": "POST",
            "url": "/services/data/v62.0/sobjects/Order__c",
            "body": { "Name": "Order 123" }
        },
        // 2. Create child second, reference parent
        {
            "referenceId": "item1",
            "method": "POST",
            "url": "/services/data/v62.0/sobjects/Order_Item__c",
            "body": {
                "Order__c": "@{order1.id}", // Now valid
                "Item__c": "itemId"
            }
        }
    ]
}
```

2. **See composite-request-order.md** for complete operation sequencing.

## Integration Best Practices

### Error Handling Strategy

1. **Always check allOrNone**:
```javascript
{
    "allOrNone": false // Continue processing if one fails
}
```

2. **Include detailed error logging**:
```javascript
// Tray error handler
if (response.compositeResponse[0].httpStatusCode !== 201) {
    const error = response.compositeResponse[0].body[0];
    console.error(`Salesforce error: ${error.errorCode} - ${error.message}`);
}
```

3. **Implement retry logic** for transient failures:
```javascript
// Retry with exponential backoff
const maxRetries = 3;
for (let i = 0; i < maxRetries; i++) {
    try {
        return await executeSalesforceRequest();
    } catch (err) {
        if (i === maxRetries - 1) throw err;
        await sleep(Math.pow(2, i) * 1000);
    }
}
```
